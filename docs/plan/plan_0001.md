# 최종 구현 계획: AI-GEN Image Detector

## 1. 문서 목적과 범위

이 문서는 `docs/draft/draft_0001.md`를 검수한 뒤 작성한 최종 구현 계획이다. 산출물은 실행 코드가 아니라, 이후 코딩 에이전트가 현재 저장소 루트에서 구현을 진행하기 위한 절차와 계약이다.

현재 저장소는 문서 중심 상태로 간주한다. 구현 전 반드시 저장소 루트를 감사하고, 기존 코드가 없다면 현재 루트에 직접 `src/`, `scripts/`, `app/`, `configs/`, `tests/`, `outputs/`를 만든다. 특별한 이유 없이 중첩 디렉터리 `ai-gen-image-detector/`를 새로 만들지 않는다.

프로젝트의 핵심 목표는 사용자가 JPG/PNG 이미지를 입력하면 `real` 또는 `AI-generated` 확률을 출력하는 실험용 웹 탐지기를 만드는 것이다. 최종 구현은 frozen `openCLIP ViT-B/32` 이미지 특징, FFT/DCT hand-crafted frequency 특징, feature cache, CLIP-only/frequency-only/fusion 실험, `LogisticRegression` 및 `Linear SVM`, robustness test, Streamlit demo, README를 포함해야 한다.

Phase A는 빠른 smoke/integration 단계이다. Phase A의 frequency-only `LogisticRegression` 결과는 최종 성공이 아니다. 최종 draft-compliance는 Phase B 완료 기준으로 판단한다.

## 3. 최종 Acceptance Criteria

최종 acceptance는 Phase B 기준이다.

- 현재 저장소 루트에서 `python scripts/...` 형식의 CLI가 import 오류 없이 실행된다.
- CIFAKE-style local subset을 deterministic manifest로 만들 수 있다.
- manifest는 `real=0`, `fake=1` label polarity와 split, path, sample ID를 검증한다.
- CLIP, frequency, fusion cache는 cache v1 schema를 통과한다.
- `CLIP-only`, `frequency-only`, `fusion` 각각에 대해 `LogisticRegression` 결과가 생성된다.
- `Linear SVM`은 calibrated probability를 제공하거나 decision-score-only 제한을 config/README/artifact에 명시한다.
- 모든 `prob_fake`는 `P(fake=1)` 의미를 유지한다.
- scaler, PCA, calibration은 train split만 사용한다. threshold 선택은 validation split만 사용한다.
- test와 robustness split은 reporting only이며 refit에 사용하지 않는다.
- 각 실험 디렉터리에 `metrics.json`, `predictions.csv`, `confusion_matrix.png`, `roc_curve.png`, `pr_curve.png`, model artifact, scaler artifact, config snapshot이 존재하고 내용 검증을 통과한다.
- robustness는 같은 `base_sample_id` 집합에 corruption을 적용하고 feature를 재추출하며 clean scaler/classifier를 재사용한다.
- Streamlit demo는 saved predictor config로 JPG/PNG를 처리하고 `prob_fake`, threshold 결과, 한계점 warning을 표시한다.
- README는 설치, 데이터 준비, feature 추출, 학습, 평가, robustness, Streamlit 실행, 한계점을 포함한다.
- 정확도/AUC의 고정 목표값은 acceptance로 만들지 않는다. finite metric, non-leaky computation, artifact completeness를 기준으로 한다.

## 4. Phase A/B/C 구현 범위와 Exit Gate

### Phase A: smoke/integration

목적은 CLIP/GPU/network 없이 data → manifest → cache → classifier → metrics 경로를 증명하는 것이다.

구현 범위:

- 저장소 감사 및 현재 루트 scaffold
- import/script 실행 정책
- `requirements.txt`, 기본 config
- tiny synthetic/local fixture
- local CIFAKE-style loader
- manifest v1 generator/validator
- compact FFT/DCT frequency extractor
- cache v1 writer/reader/validator
- frequency-only `LogisticRegression`
- `metrics.json`, `predictions.csv` smoke artifact
- model/scaler reload equivalence check
- no-download unit tests, CPU quick test

Exit gate:

- `python scripts/prepare_cifake_subset.py --help`가 동작한다.
- tiny fixture에서 manifest 생성, frequency cache 생성, LR 학습/평가가 CPU에서 완료된다.
- cache schema, feature finite check, label polarity, model reload check가 통과한다.
- Phase A 결과는 최종 acceptance로 보고하지 않는다.

### Phase B: draft-compliance core

목적은 원문 핵심 요구사항을 충족하는 최소 완성 파이프라인을 만드는 것이다.

구현 범위:

- real CIFAKE subset 준비 경로
- exact openCLIP model smoke: `hf-hub:laion/CLIP-ViT-B-32-laion2B-s34B-b79K`
- CLIP feature cache
- frequency feature cache
- CLIP-only/frequency-only/fusion `LogisticRegression`
- `Linear SVM` handling
- branch-wise normalization 및 fusion alignment
- full metrics/plots/artifacts
- robustness quick/full runner
- `run_all_experiments.py --quick`와 full mode
- `src/inference/predictor.py` artifact loading contract
- Streamlit demo
- README와 한계점
- 접근 가능 시 remote CUDA smoke/full evidence

Exit gate:

- 세 실험 mode의 LR 결과가 생성된다.
- `Linear SVM`의 probability 또는 decision-score 제한이 명확하다.
- robustness CSV와 degradation이 생성된다.
- Streamlit이 saved artifact로 단일 이미지 추론을 수행한다.
- README만 보고 Phase A와 Phase B quick path를 따라갈 수 있다.

### Phase C: stretch/deferred

다음은 silent omission이 아니라 명시적 보류 항목이다.

- MLP training
- HF loader hardening
- RBF SVM
- UMAP
- advanced FFT peak prominence
- polished Streamlit selector/UX
- GenImage-compatible extension skeleton
- `pyproject.toml`
- Linear SVM calibrated probability가 Phase B에서 빠진 경우 calibration 강화

## 6. 데이터셋 및 Manifest 계약

manifest는 모든 feature extraction, training, evaluation, robustness의 단일 source of truth이다.

필수 columns:

| column | 의미 |
|---|---|
| `sample_id` | clean sample의 안정 ID |
| `base_sample_id` | corrupted/derived sample이면 원본 ID, clean이면 빈 값 또는 자기 자신 |
| `rel_path` | `data_root` 기준 POSIX relative path |
| `root` | manifest가 참조하는 data root |
| `label` | numeric label, `real=0`, `fake=1` |
| `class_name` | `real` 또는 `fake` |
| `source` | `cifake_local`, `cifake_hf`, `fixture` 등 |
| `source_split` | 원본이 train/test를 제공하면 기록 |
| `split` | project split: `train`, `val`, `test` |
| `width`, `height` | decode 후 이미지 크기 |
| `sha256` | 가능하면 원본 file hash |
| `file_size` 또는 `mtime` | hash 생략 시 최소 file identity 보조 정보 |
| `status` | `ok`, `missing`, `corrupt`, `duplicate` 등 |

규칙:

- positive class는 항상 `fake=1`이다.
- `sample_id`는 deterministic sorted input과 normalized relative path를 기준으로 생성한다.
- 같은 byte hash가 train/val/test에 중복되면 leakage risk로 기록하고 기본적으로 실패시킨다.
- corrupt/missing image는 manifest validation에서 명확히 보고한다.
- 이미 train/test가 있는 원본 구조는 `source_split`에 보존한다. project split은 별도 `split`에 저장한다.
- 기본 split은 class-balanced 70:15:15이며 seed는 config에 저장한다.
- subset은 copy, symlink, manifest-only 중 하나로 명시한다. 기본은 copy 또는 manifest-only이며, scripts가 같은 방식을 공유해야 한다.
- 모든 배열은 manifest order로 반환한다.

## 7. 이미지 전처리 계약

공통:

- PIL로 image를 열 때 EXIF orientation을 적용한다.
- 모든 입력은 RGB로 변환한다.
- 실패한 image는 manifest `status`와 error log에 남긴다.

CLIP branch:

- `open_clip.create_model_from_pretrained("hf-hub:laion/CLIP-ViT-B-32-laion2B-s34B-b79K")`에서 제공하는 preprocess를 사용한다.
- feature extraction은 `model.eval()`, `torch.no_grad()`, frozen parameter 상태에서 수행한다.
- output feature는 per-sample L2 normalization을 적용할 수 있으며 cache metadata에 `normalization="clip_l2"`로 기록한다.

Frequency branch:

- RGB에서 luminance 또는 grayscale 변환 방식을 하나로 고정한다.
- 기본 frequency input size는 224x224로 고정한다. 변경 시 config와 cache invalidation 대상이다.
- FFT는 centered spectrum, `log(abs(fft) + epsilon)` 정책을 metadata에 기록한다.
- radial bins 기본값은 64이다.
- DCT는 whole-image DCT와 8x8 block DCT 통계를 포함한다.
- image size가 8의 배수가 아니면 padding 또는 cropping 정책을 고정한다.

Robustness:

- JPEG/resize/blur corruption은 원본 test image에 먼저 적용한다.
- corruption 후 branch별 preprocess와 feature extraction을 다시 수행한다.
- clean feature를 재사용해 robustness를 계산하지 않는다.

## 8. Feature Cache v1 계약

cache는 하나의 `.pt` dict로 저장한다.

필수 keys:

```python
{
    "schema_version": "feature_cache_v1",
    "manifest_hash": "...",
    "feature_type": "clip" or "frequency",
    "feature_config_hash": "...",
    "sample_ids": list[str],
    "paths": list[str],
    "labels": torch.Tensor or np.ndarray,
    "splits": list[str],
    "features": torch.Tensor or np.ndarray,
    "metadata": {...},
}
```

metadata 필수 fields:

- `feature_dim`
- `dtype`
- `normalization`
- `label_mapping`: `{"real": 0, "fake": 1}`
- CLIP cache: `model_name`, `preprocess_hash`, `device`
- frequency cache: `image_size`, `radial_bins`, `fft_epsilon`, `dct_policy`
- `created_at`
- `seed`

corrupted cache metadata:

- `base_sample_ids`
- `corruption_type`
- `corruption_level`
- `clean_manifest_hash`

validator 필수 규칙:

- `N == len(sample_ids) == len(paths) == len(labels) == len(splits) == features.shape[0]`
- feature는 finite value만 포함해야 한다.
- `feature_dim`과 실제 shape가 일치해야 한다.
- `sample_ids`, path, label, split은 manifest와 일치해야 한다.
- manifest order 기준으로 alignment해야 한다.
- duplicate/missing/mismatched ID는 실패한다.
- manifest, label mapping, model name, CLIP preprocess, frequency config, radial bins, image size, corruption settings, normalization policy가 바뀌면 cache를 reject하거나 namespace를 분리한다.

## 9. Feature Extractor 구현 계약: CLIP / FFT / DCT

CLIP extractor:

- batch inference를 지원한다.
- device는 CLI argument로 받는다.
- CUDA가 요청됐지만 사용할 수 없으면 명확한 error를 낸다.
- CPU smoke는 1-2 image로만 수행한다.
- model load smoke는 full extraction 전에 독립 gate로 둔다.

FFT feature:

- 64-bin radial log power spectrum
- low/mid/high frequency energy
- high-frequency energy ratio
- low/high energy ratio
- spectral slope
- peak count는 optional 또는 Phase C

DCT feature:

- whole-image DCT coefficient statistics
- 8x8 block-wise DCT statistics
- low/mid/high coefficient energy
- coefficient mean/std/max
- high-frequency DCT energy ratio

최종 frequency feature dimension은 100-250 범위로 유지한다.

## 10. 학습, 정규화, 확률, Model Artifact 계약

지원 matrix:

| classifier | 단계 | probability 정책 | Streamlit 사용 |
|---|---:|---|---|
| `LogisticRegression` | A/B | `predict_proba` | 가능 |
| `Linear SVM` | B | calibrated probability 또는 decision score | probability 없으면 Streamlit probability model로 사용 금지 |
| MLP | C | sigmoid/logits 명시 | 구현 시만 가능 |
| RBF SVM | C | optional | 기본 제외 |

정규화와 leakage:

- CLIP feature는 per-sample L2 normalization 가능.
- frequency raw feature를 cache하고, `StandardScaler`는 train split으로만 fit한다.
- fusion은 branch별 정규화 후 concat한다.
- concat 후 추가 scaler를 쓸 경우 train split으로만 fit한다.
- calibration, PCA, threshold 선택은 test를 사용하지 않는다.
- threshold 기본값은 0.5이며, val-selected threshold를 쓰면 config에 저장한다.
- `prob_fake`는 항상 `P(fake=1)`이다.

model artifact 필수 구성:

```text
outputs/experiments/{experiment_name}/
  model.joblib or model.pt
  scaler.joblib
  config.yaml
  metrics.json
  predictions.csv
  confusion_matrix.png
  roc_curve.png
  pr_curve.png
```

config에는 다음을 저장한다.

- `mode`: `clip_only`, `frequency_only`, `fusion`
- classifier type
- calibration status
- threshold
- label mapping
- feature config
- scaler path
- manifest/cache hash
- training command
- seed
- package/device metadata

`src/inference/predictor.py`와 Streamlit은 이 artifact만으로 reload해야 한다.

## 11. 평가 지표와 결과 Artifact 계약

필수 metrics:

- Accuracy
- Precision
- Recall
- F1
- ROC-AUC
- Average Precision 또는 PR-AUC
- Confusion matrix

`predictions.csv` 필수 columns:

```text
path,sample_id,label,pred_label,prob_fake,score,split
```

규칙:

- ROC-AUC와 AP는 continuous score 또는 probability로 계산한다.
- confusion matrix, accuracy, precision, recall, F1은 저장된 threshold로 계산한다.
- tiny smoke에서 class가 하나뿐이면 metric validity test가 아니라 shape/command smoke로만 처리한다.
- Phase B metric run은 eval split에 두 class가 모두 있어야 한다.
- plot file은 존재뿐 아니라 non-empty 여부를 확인한다.

## 12. Robustness Test 계약

quick mode:

- tiny test subset
- corruption family별 1개 level
- CPU 또는 작은 CLIP smoke로 mechanism 확인

full mode:

- JPEG quality: 95, 75, 50, 30
- resize: 224 -> 160 -> 224, 224 -> 128 -> 224
- Gaussian blur sigma: 0.5, 1.0, 2.0

공통 규칙:

- 동일한 clean test `base_sample_id` 집합을 사용한다.
- corruption은 image에 적용한 뒤 feature를 재추출한다.
- clean-trained scaler/classifier를 재사용한다.
- corrupted feature에서 scaler나 classifier를 refit하지 않는다.
- clean metric, corrupted metric, `clean - corrupted` degradation을 저장한다.

필수 산출물:

```text
outputs/robustness/{experiment_name}/
  robustness_metrics.csv
  robustness_summary.png
```

## 16. 원격 RTX5090 실행 및 rsync 계획

remote 정보:

```text
host: tml-server
remote path: ~/codes/ml-project
conda env: ml_termproj
```

rsync 원칙:

- 기본적으로 source code, configs, scripts, app, tests, README만 sync한다.
- `data/`, `outputs/`, virtualenv, cache는 의도하지 않으면 제외한다.
- 대용량 data와 outputs는 별도 명령으로 관리한다.

예시 절차:

```bash
rsync -av --exclude data --exclude outputs --exclude .venv --exclude __pycache__ ./ tml-server:~/codes/ml-project/
ssh tml-server
conda activate ml_termproj
python scripts/extract_clip_features.py --data_root data/cifake_subset --output_path outputs/features/clip_smoke.pt --batch_size 2 --device cuda --max_samples 2
```

수집 evidence:

- command log
- `torch.__version__`
- CUDA availability와 device name
- `open_clip_torch` version
- cache metadata
- metrics/artifact list

이 계획은 remote 실행이 이미 완료됐다고 주장하지 않는다.

한계점 필수 문구:

- DALL-E, Midjourney, SDXL, FLUX 등에 일반화하려면 GenImage, Synthbuster, ForenSynths 등 외부 데이터셋 평가가 필요하다.

## 18. 보류/제외 항목과 리스크

보류:

- MLP: overfitting, checkpoint, calibration 복잡도 때문에 Phase C
- HF loader hardening: schema 확인 전까지 Phase C
- RBF SVM: optional
- UMAP: optional
- advanced peak prominence: optional
- GenImage: 전체 다운로드 금지, extension note만 허용
- polished Streamlit selector: artifact가 쌓인 뒤 Phase C
- `pyproject.toml`: 필요 시 추가

리스크:

- CIFAKE data source 부재
- CLIP model download 실패
- remote credential 부재
- CUDA package mismatch
- stale cache reuse
- label inversion
- train/test leakage
- robustness runtime 폭증

각 리스크는 README 또는 run log에 blocker로 남긴다. quick synthetic test가 full Phase B evidence를 대체하지 않는다.

## 19. 코딩 에이전트 주의사항

- Phase A 결과를 최종 성공으로 보고하지 않는다.
- manifest 없이 directory walk 결과를 바로 학습하지 않는다.
- CLIP/frequency cache를 row order만 믿고 concat하지 않는다.
- `sample_id` alignment 실패를 자동 무시하지 않는다.
- stale cache를 재사용하지 않는다.
- scaler, PCA, calibration, threshold 선택에 test split을 쓰지 않는다.
- robustness에서 corrupted data로 scaler/classifier를 refit하지 않는다.
- Streamlit에 존재하지 않는 model mode를 노출하지 않는다.
- CIFAKE 성능을 real-world detector 성능으로 과장하지 않는다.
- generic experiment registry, plugin framework, multi-backbone framework를 만들지 않는다.
