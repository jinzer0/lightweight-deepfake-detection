# AI-GEN Image Detector 프로젝트 구현 지시 프롬프트

너는 Python/PyTorch 기반 머신러닝 프로젝트를 구현하는 코딩 에이전트다.
현재 프로젝트는 **AI-GEN Image Detector**이며, 목표는 사용자가 이미지를 입력했을 때 해당 이미지가 실제 이미지인지 AI 생성 이미지인지 분류하고, 모델별 성능 비교 및 주파수 기반 설명 정보를 제공하는 것이다.

## 0. 실행 환경 고정

반드시 아래 conda 환경을 사용한다.

```bash
conda activate ml_termproj
```

## 1. 핵심 구현 조건

아래 조건은 반드시 지킨다.

1. `CIFAKE`는 사용하지 않는다.
2. 데이터셋은 **GenImage**를 사용한다.
3. CLIP feature extractor는 반드시 `open_clip_torch`를 사용한다.
4. CLIP 모델은 반드시 아래 모델을 사용한다.

```python
laion/CLIP-ViT-L-14-laion2B-s32B-b82K
```

가능하면 다음 방식으로 로드한다.

```python
import open_clip

model, preprocess = open_clip.create_model_from_pretrained(
    "hf-hub:laion/CLIP-ViT-L-14-laion2B-s32B-b82K"
)
```

만약 위 방식이 현재 환경에서 실패하면 fallback으로 다음을 사용한다.

```python
model, _, preprocess = open_clip.create_model_and_transforms(
    "ViT-L-14",
    pretrained="laion2B-s32B-b82K"
)
```

CLIP encoder는 기본적으로 **freeze**한다. CLIP fine-tuning은 하지 않는다.

## 2. 구현해야 하는 모델

최소한 아래 네 가지 모델을 모두 구현하고 성능 비교가 가능해야 한다.

### 2.1 Baseline: ResNet-50

* `torchvision.models.resnet50` 사용
* ImageNet pretrained weight 사용
* 마지막 `fc`를 binary classification용으로 교체
* 입력 크기: 512x512
* 출력: AI-generated probability
* loss: `BCEWithLogitsLoss`
* metric: Accuracy, Precision, Recall, F1, ROC-AUC, Confusion Matrix

### 2.2 CLIP-only

* `open_clip_torch`로 CLIP image embedding 추출
* CLIP encoder는 frozen
* image embedding은 L2 normalize
* feature cache를 `.pt` 또는 `.npy`로 저장
* classifier는 최소한 다음 중 하나 이상 구현

  * Logistic Regression
  * shallow MLP
* 출력: AI-generated probability

### 2.3 Frequency-only

입력 이미지에서 주파수 도메인 feature를 추출하고, 해당 feature만으로 binary classifier를 학습한다.

필수 feature:

* DCT 기반 radial spectrum
* FFT 기반 radial spectrum
* high-frequency ratio
* low/mid/high band energy ratio
* block DCT statistics

  * mean
  * std
  * max
  * selected quantile values

주의 사항:

* train split에서만 scaler를 fit한다.
* validation/test에는 train scaler를 transform만 적용한다.
* `StandardScaler`를 저장한다.
* classifier는 Logistic Regression, SVM, MLP 중 최소 1개 이상 구현한다.
* 가능하면 Logistic Regression과 MLP 둘 다 구현한다.

### 2.4 Fusion model

CLIP feature와 frequency feature를 concat하여 fusion classifier를 학습한다.

입력:

```text
fusion_input = concat(
    normalized_clip_embedding,
    standardized_frequency_feature
)
```

Fusion classifier는 반드시 **15~20개 MLP layer** 수준의 깊은 구조로 구현한다. 단순 Linear 한두 개로 끝내지 않는다.

권장 구조:

```text
Input LayerNorm
Input projection Linear
Residual MLP Block x 8
  - 각 block은 Linear -> GELU -> Dropout -> Linear -> Dropout
  - residual connection 적용
  - LayerNorm 적용
Final LayerNorm
Final Linear -> logit
```

8개 residual block이면 block 내부 Linear가 총 16개이므로 “15~20개 MLP layer” 조건을 만족한다.

일반화 성능 강화를 위해 아래 기법을 적용한다.

* Dropout
* Residual connection
* LayerNorm
* AdamW
* weight decay
* early stopping
* learning rate scheduler
* gradient clipping
* validation ROC-AUC 기준 best checkpoint 저장
* seed 고정
* class imbalance가 있으면 `pos_weight` 또는 weighted sampler 적용

Fusion model은 최종 메인 모델로 사용한다.

## 3. GenImage 데이터셋 처리

GenImage root path는 하드코딩하지 않는다.
모든 스크립트에서 `--data_root` 또는 config yaml로 입력받는다.

예시:

```bash
python scripts/build_manifest.py \
  --data_root /path/to/GenImage \
  --out data/genimage_manifest.csv
```

manifest CSV에는 최소한 아래 컬럼을 포함한다.

```text
path,label,class_name,generator,source_split,split
```

라벨 정의:

```text
real/nature = 0
ai/fake/generated = 1
```

GenImage 폴더 구조가 조금 달라도 동작하도록 recursive scan 방식으로 구현한다.

path 안에 다음 키워드가 있으면 label을 추론한다.

```text
real keywords: nature, real, natural
fake keywords: ai, fake, generated, synth, synthetic
```

generator는 가능하면 GenImage의 상위 generator 폴더명에서 추출한다.

예시 generator:

```text
stable_diffusion
midjourney
biggan
glide
adm
wukong
vqdm
```

실제 폴더명이 다르면 코드가 자동으로 top-level folder명을 generator로 잡게 만든다.

## 4. Split 전략

두 가지 split mode를 구현한다.

### 4.1 random split

```text
train:val:test = 70:15:15
```

class-balanced stratified split 적용.

### 4.2 generator_holdout split

일반화 평가를 위해 generator 단위 holdout split을 지원한다.

예시:

```bash
python scripts/build_manifest.py \
  --data_root /path/to/GenImage \
  --out data/genimage_manifest.csv \
  --split_mode generator_holdout \
  --holdout_generators midjourney,glide
```

요구사항:

* train generator와 test generator가 겹치면 안 된다.
* validation은 train generator 내부에서 class-balanced로 분리한다.
* per-generator sample count를 출력한다.
* 데이터 누수 방지를 위해 동일 파일 중복 여부를 검사한다.
* 가능하면 파일 hash 기반 중복 검사 옵션을 구현한다.

## 11. Confidence level 기준

기본 기준은 다음처럼 구현하되 config에서 수정 가능하게 한다.

```text
prob >= 0.8 또는 prob <= 0.2: 높음
0.65 <= prob < 0.8 또는 0.2 < prob <= 0.35: 중간
0.35 < prob < 0.65: 낮음
```

최종 판정:

```text
prob >= threshold: AI-generated
prob < threshold: Real
```

기본 threshold는 0.5로 두고, validation set에서 best F1 또는 Youden’s J 기준 threshold를 별도로 계산하여 저장한다.

## 16. 작업 방식

먼저 현재 repository 구조를 확인하고, 누락된 부분만 구현한다.
구현 후에는 작은 GenImage subset으로 smoke test가 가능하도록 `--max_samples_per_generator` 옵션을 제공한다.

예시:

```bash
python scripts/build_manifest.py \
  --data_root /path/to/GenImage \
  --out data/genimage_manifest.csv \
  --max_samples_per_generator 200
```

대규모 학습 전에 반드시 small subset smoke test를 통과시킨다.

가짜 성능 수치를 만들지 말고, 실제 실행 결과가 없으면 “not run”으로 표시한다.
